3. Follow [SETUP.md](SETUP.md) or [QUICK_START_CHECKLIST.md](QUICK_START_CHECKLIST.md)
4. Get system running (Backend + Frontend + MongoDB)

1. Read [MERN_LEARNING_GUIDE.md](MERN_LEARNING_GUIDE.md)
2. Follow [NEXT_STEPS.md](NEXT_STEPS.md) - Create sample data
3. Test Purchase Order workflow → GRN workflow
4. Test Sales Order workflow
5. Check low stock alerts

3. Study data relationships
4. Check API_DOCUMENTATION.md for endpoints

1. Read [GITHUB_SETUP.md](GITHUB_SETUP.md)
2. Push to GitHub
3. Verify code is on GitHub
4. Share link

### Day 5: Interview Prep
1. Read [INTERVIEW_GUIDE.md](INTERVIEW_GUIDE.md)
2. Practice explaining the system
3. Be ready to show GitHub
4. Prepare to answer technical questions
